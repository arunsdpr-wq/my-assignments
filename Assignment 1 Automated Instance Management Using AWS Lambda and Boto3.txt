Step 1: Prepare Your EC2 Instances 🖥️
First, we need the "targets" for our script.

Launch two t2.micro instances (Amazon Linux 2023).

Go to the Tags tab for each instance.

Instance A: Add Key: Action, Value: Auto-Stop.

Instance B: Add Key: Action, Value: Auto-Start.

Ensure Instance A is Running and Instance B is Stopped so we can actually see the Lambda do its magic.

Step 2: Create the IAM "Permission Slip" 
Lambda needs your "permission slip" to touch other AWS resources.

Navigate to the IAM Dashboard > Roles > Create role.

Select AWS Service and choose Lambda.

Search for and attach the policy: AmazonEC2FullAccess.

Name it something like LambdaEC2ManagerRole and create it.

Step 3: The Lambda Function & Boto3 Script 
Now for the brains of the operation.

Go to Lambda > Create function.

Name it EC2AutoManager. Select Python 3.x as the runtime.

Under Change default execution role, select "Use an existing role" and pick the one you made in Step 2.

In the Code tab, replace the default code with this script:

-------------------------------------
import boto3

def lambda_handler(event, context):
    # 1. Initialize the EC2 client
    ec2 = boto3.client('ec2')
    
    # 2. Define filters to find instances by our specific tags
    stop_filter = [{'Name': 'tag:Action', 'Values': ['Auto-Stop']}]
    start_filter = [{'Name': 'tag:Action', 'Values': ['Auto-Start']}]
    
    # 3. Find instances to STOP
    stop_instances = ec2.describe_instances(Filters=stop_filter)
    stop_ids = []
    for reservation in stop_instances['Reservations']:
        for instance in reservation['Instances']:
            stop_ids.append(instance['InstanceId'])
            
    # 4. Find instances to START
    start_instances = ec2.describe_instances(Filters=start_filter)
    start_ids = []
    for reservation in start_instances['Reservations']:
        for instance in reservation['Instances']:
            start_ids.append(instance['InstanceId'])
            
    # 5. Execute Actions
    if stop_ids:
        ec2.stop_instances(InstanceIds=stop_ids)
        print(f"✅ Successfully requested STOP for: {stop_ids}")
    else:
        print("ℹ️ No instances found with 'Auto-Stop' tag.")
        
    if start_ids:
        ec2.start_instances(InstanceIds=start_ids)
        print(f"✅ Successfully requested START for: {start_ids}")
    else:
        print("ℹ️ No instances found with 'Auto-Start' tag.")

    return {
        'statusCode': 200,
        'body': 'EC2 Management Task Completed!'
    }
--------------------------------------

Step 4: Testing & Verification ✅
Click Deploy to save your code.

Click Test, give the test event a name myTestEvent, and hit Save.

Click Test again to run it.

In the Configuration Tab set timeout 30 sec. otherwise it will show error of timeout.

The Reveal: Go back to your EC2 Dashboard. You should see Instance A entering the "Stopping" state and Instance B entering the "Pending/Running" state.
